# MCP в Vanessa Automation

## Быстрый старт

Для начала использования MCP в Vanessa Automation выполните следующие шаги:

1. **Настройте LLM с поддержкой MCP**. У вас должна быть работающая LLM, к которой можно подключить MCP инструменты. Например, VSCode + плагин Cline — это среда, где вы пишете промты.

2. **Установите расширение в базу менеджера тестирования**. В базу менеджера тестирования необходимо поставить расширение из [проекта](https://github.com/1c-neurofish/onec-client-mcp-devkit/releases).

3. **Запустите MCP сервер**. Нажмите на кнопку **Запустить** в расширении.

4. **Настройте MCP клиент в вашей IDE**. В вашей IDE, где работает LLM, необходимо прописать настройки MCP сервера, который запущен в менеджере тестирования. В инструкции ниже приведены примеры настроек для плагина Continue (для VSCode), плагина Cline и плагина Roo Code (тоже для VSCode).

5. **Включите внешнюю компоненту VanessaExt**. В Vanessa Automation необходимо включить использование внешней компоненты VanessaExt.

6. **Установите расширение VAExtension**. В базу клиента тестирования необходимо добавить расширение VAExtension.

7. **Настройка завершена**. После выполнения всех шагов MCP готов к работе.

## Пример запуска сеанса 1С из командной строки так, чтобы сразу запустился MCP сервер
```batch
"C:\Program Files\1cv8\8.3.27.2130\bin\1cv8c.exe" ENTERPRISE /TESTMANAGER /IBConnectionString "File=""c:\Bases\8.3.27"";" /Execute "C:\Commons\rep\vanessa-automation\vanessa-automation.epf" /C "runMcp;mcpPort=9876"

```

## Доступные инструменты:

Ниже приведена таблица всех доступных MCP инструментов Vanessa Automation, отсортированная по имени инструмента.

| Инструмент | Описание |
|------------|----------|
| **check_syntax** | Проверяет синтаксис Gherkin в активном документе редактора Vanessa Automation или в указанном feature-файле. Возвращает Markdown с найденными ошибками: неизвестные шаги, структурные проблемы, ошибки ключевых слов. |
| **close_test_client** | Закрывает сеанс клиента тестирования по имени профиля. Если профиль не указан — закрывает текущий подключенный профиль. Используйте для завершения работы с клиентом тестирования после выполнения тестов. |
| **connect_test_client** | Подключает клиент тестирования 1С по имени профиля. Необходимо для работы **get_form_analysis**, **execute_feature_step** и запуска сценариев, которым требуется клиент тестирования для работы. |
| **execute_feature_step** | Выполняет шаг из .feature файла по номеру строки. Валидирует что файл открыт в редакторе, активирует вкладку при необходимости. |
| **execute_form_actions** | Выполняет последовательность действий с элементами формы клиента тестирования. Позволяет выполнить несколько действий за один вызов инструмента. Параметр **actions_json** (обязательный) - JSON-строка с массивом действий. Каждое действие в массиве содержит: **action** - тип действия (click_button, input_text, и т.д.), **element_name** - имя элемента формы, **table_name** - имя таблицы (если элемент в таблице), **value** - значение для ввода/выбора, другие параметры в зависимости от действия. При возникновении ошибки выполнение прерывается. Требует подключённого клиента тестирования. |
| **frequently_used_steps** | Возвращает информацию о том какие шаги чаще используются в статистике больших проектов. В начале идут часто используемые шаги, далее идут более редко используемые шаги. Возвращает представление шага, описание шага, тип шага. Также возвращает информацию сколько всего шагов есть в накопленной статистике. |
| **get_active_window_data** | В зависимости от параметра **type** возвращает важные свойства активного окна клиента тестирования: если type=**window_caption** - возвращает заголовок окна. По заголовку окна можно найти окно и получить его данные, можно активизировать окно, можно закрыть окно. Если type=**navigation_link** - возвращает навигационную ссылку активного окна. Навигационная ссылка потом может быть использована для открытия окна по навигационной ссылке. Также навигационная ссылка помогает понять с каким объектом метаданных связано активное окно. Если type=**form_name** - возвращает имя формы, подчиненной активному окну. Имя формы позволяет понять с каким объектом метаданных связано активное окно, т.к. в имени формы чаще всего содержится имя объекта метаданных. Если type=**form_caption** - возвращает заголовок формы, подчиненной активному окну. Чаще всего такой заголовок формы совпадает с заголовком окна-владельца и он нужен редко. |
| **get_data_from_knowledge_base** | Получает информацию из базы знаний, в которой хранится информация о часто возникающих вопросах и как их решить. Данные в базе знаний хранятся в формате Вопрос - ответ. Чтобы экономить токены можно получать данные порциями. Можно получить только тексты вопросов, чтобы потом получить ответ только на нужный вопрос. Можно выполнить поиск по вопросам и ответам. Поиск работает без учёта регистра букв. |
| **get_editor_state** | Возвращает состояние Monaco-редактора Vanessa Automation: открытые вкладки, содержимое активного документа, позицию курсора, ошибки. Один вызов для полного понимания контекста редактора. Если feature файл загружен в редактор Vanessa Automation, то уже можно запускать его сценарии на выполнение с помощью **run_scenario**. |
| **get_environment_data** | Получает важные данные об окружении: текущая дата и время, версия платформы 1С:Предприятие, данные об операционной системе, какая используется VA - обычная или Single, включено ли использование внешней компоненты VanessaExt, в какой базе запущен менеджер тестирования. |
| **get_extension_list** | Получает список всех расширений, установленных в клиенте тестирования. Клиент тестирования должен быть подключен. Возвращает информацию о каждом расширении: Имя, Версия, Назначение; Флаги: Активно, БезопасныйРежим. |
| **get_form_analysis** | Возвращает информацию о текущей форме тестируемого приложения: дерево элементов или состояние формы как Gherkin-шаги. Требует подключённого клиента тестирования. Важно обращать на флаг Видимости у элемента формы и групп, в которые он входит. Если у элемента Видимость=Истина, а у группы Видимость=Ложь, то скорее всего элемент формы не виден пользователю и он не может с ним взаимодействовать. |
| **get_form_element_data** | Получает данные указанного элемента формы клиента тестирования. Возвращает информацию об элементе в формате Markdown: Имя элемента, Заголовок, Тип, Вид, Значение, Представление (если применимо), Флаги: Видимость, Доступность, ТолькоПросмотр. Параметр **element_name** обязателен и должен содержать имя элемента формы. |
| **get_info_about_line_scenario** | Получает подробную информацию о указанной строке файла открытого в редакторе. Если в строке находится шаг сценария, то будет возвращена подробная информация о шаге. |
| **get_object_attributes** | Получает значения реквизитов объекта (и их тип), связанного с открытой формой. Работает только для форм объектов (Справочники, Документы и т.п.), не работает для форм списков и служебных форм. Параметр **data_mode** определяет состав возвращаемых данных: 'all' (все данные - шапка и все табличные части), 'attributes_only' (только реквизиты шапки), 'single_attribute' (значение указанного реквизита шапки, требуется **attribute_name**), 'tabular_sections_only' (только все табличные части), 'single_tabular_section' (только указанная табличная часть, требуется **tabular_section_name**). Параметр **only_filled** определяет фильтровать ли пустые реквизиты: true (только заполненные), false (все реквизиты, по умолчанию). Возвращает данные в формате Markdown с именами, типами и значениями реквизитов. |
| **get_table_data** | Получает данные из таблицы базы данных. Цель - понять какие тестовые данные уже есть в базе, чтобы лучше понимать какими данными заполнять объекты в тестах. Нужно указать Тип - **object_type**: 1 = Справочник, 2 = Документ, 3 = Перечисление, 4 = Константа, 5 = Реквизиты объекта по навигационной ссылке. Надо указать Вид - **object_name** (обязательный параметр для object_type 1-4). **limit** - сколько элементов возвращать (необязательный, по умолчанию 10). Для справочников будет возвращено Наименование, Код, Владелец (если такие поля есть). Если справочник подчинен другому справочнику, то можно указать владельца через параметр **owner**. Для документа будут возвращены Дата, Номер. Для справочников и документов будет также возвращена навигационная ссылка. Для документов можно передать **date_start** и **date_end** (оба необязательные, формат: "30.12.2025 15:45:55"). **name_filter** - отбор по наименованию элемента (необязательный, работает для Справочников). **navigationLink** - навигационная ссылка объекта для получения реквизитов (обязательный параметр для object_type=5). |
| **get_test_results** | Подробные результаты выполнения сценария: статус каждого шага, время, ошибки. Если scenarioId не указан — результат последнего сценария. |
| **get_VanessaAutomation_state** | Возвращает текущее состояние Vanessa Automation в формате Markdown: выполняется ли сценарий, текущая фича (имя, путь, язык, теги), текущий сценарий (имя, статус, теги), текущий шаг (текст, статус, ошибка), подключен ли клиент тестирования. Используйте для получения контекста перед написанием или анализом тестов. |
| **get_window_list_os** | Возвращает список окон доступных для снятия скриншота с точки зрения операционной системы. Нужно вызвать перед **get_window_screenshot_os**. |
| **get_window_list_testclient** | Получает список внутренних окон клиента тестирования. Результат можно использовать для управления окнами с помощью инструмента **window_management** (активизировать, закрыть, открыть навигационную ссылку), либо для получения данных из текущего окна **get_active_window_data**. Результат нельзя использовать для получения скриншота окна, т.к. список внутренних окон клиента тестирования и список окон с точки зрения операционной системы - это разные вещи. |
| **get_window_screenshot_os** | Делает скриншот окна клиента тестирования и возвращает изображение для последующего анализа скриншота с помощью компьютерного зрения. Параметр **window_title** определяет с какого окна нужно снять скриншот, т.к. у одного клиента тестирования может быть несколько окон. Параметр обязательный. Параметр **color_mode** определяет режим цвета: 'grayscale' (черно-белый, по умолчанию) или 'color' (цветной). Если клиент тестирования не подключён или не запущен, возвращает ошибку с подсказкой сначала вызвать **connect_test_client**. Чтобы получить список доступных окон для снятия скриншота надо вызвать **get_window_list_os**. |
| **load_features** | Загружает или перезагружает .feature файлы. Используйте после того как вы отредактировали .feature файл, чтобы перечитать файл с диска в Vanessa Automation. Это лучший способ перезагрузить текущий файл в редакторе не указывая имя файла. |
| **manage_breakpoints** | Управление точками останова в Gherkin-сценариях. Действия: **toggle** (по умолчанию), **remove_all**, **list**. |
| **manage_command_interface** | Универсальный инструмент для работы с командным интерфейсом 1С:Предприятия. Поддерживаемые действия (параметр action): **get_section_panel** (получить панель разделов), **get_function_panel** (получить панель функций), **click_section_panel** (нажать кнопку панели разделов), **click_function_panel** (нажать кнопку панели функций), **get_all** (получить весь командный интерфейс - панель разделов и для каждой кнопки панель функций). Для **click_section_panel**: **command** (обязательно) - имя кнопки для нажатия, **returnFunctionPanel** (необязательно) - вернуть содержимое панели функций после нажатия. Для **click_function_panel**: **command** (обязательно) - имя кнопки для нажатия, **group** (необязательно) - имя группы, если команды не уникальны. Требует подключённого клиента тестирования. |
| **manage_form_elements** | Универсальный инструмент для управления элементами формы клиента тестирования.
